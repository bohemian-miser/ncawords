Interactive demo

Channel B Modulates Channel A’s Kernel

In our trainable-Lenia systems, a cell’s rules can depend on the state itself: the dyn1/dynwave variants use a small network that reads the local state and composes that cell’s kernel from a basis, per cell, per step. This demo makes that mechanism visible by hand.

Channel A is a living Lenia pattern. Channel B is a control field you paint. Each cell’s update for A blends two separately trained physics according to B’s local value: where B ≈ 0 the cell runs the “hex” physics (short-range kernels, a fine lattice); where B ≈ 1 it runs the “dots” physics (long-range inhibition rings, polka dots). Nothing else changes — the strength of the second channel literally determines the kernel the first channel is stepped with, cell by cell.

This is a hand-built illustration of what dyn1 and dynwave learn end-to-end — there, the blend is a trained function of hidden state rather than a channel you paint yourself. More mini-demos like this one will follow as we pull other pieces of the trainable-Lenia machinery out into the open.

Kernels and weights are read live from a public training bucket, the same source the Lenia lab gallery uses. See the research log for how these physics were trained.
