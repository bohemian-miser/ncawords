Growing Neural Cellular Automata for Text

A reimplementation of the Distill article Growing Neural Cellular Automata in which the organisms being grown are not lizards or butterflies but text. Every character below unfolds from a single seed pixel, guided only by a tiny neural network that each cell runs on its own 3×3 neighborhood — and in the first figure, a single such network grows an entire string across one shared grid, letters and digits alike, with nothing but a six-bit code in each seed to say which character it should become.

A reimplementation of Mordvintsev et al., Growing Neural Cellular Automata (Distill, 2020) — reference code at google-research/self-organising-systems — trained from scratch on letter glyphs.

Seeds that carry their name

In the per-character demo, an A grows into an A because its entire network was trained for nothing else. The model at the top of this page drops that crutch: one network serves every character in the string, so identity has to live somewhere other than the weights — and it lives in the seed. Six hidden channels of each starting cell hold a 6-bit code — enough to name any of the 36 characters, letters and digits alike — and that is the only difference between a seed destined to become a C and one destined to become a 6: six numbers in the initial state. The update rule is the same everywhere on the grid, at every step.

Training turns that rule into a decoder. It has to read the code off the growing tissue, keep relaying the identity outward as the pattern expands, and steer local growth accordingly — while also handling collisions where neighboring letters run into each other. This is what makes “a word in one picture” possible: not independent canvases stitched side by side, but one substrate on which differently-labeled colonies grow together and settle their borders.

What is this?

An embryo starts as one cell, yet reliably ends up with a spine, ten fingers, and a face in roughly the right places. No cell is in charge of that outcome. Each one reads chemical signals from its immediate neighbors, consults the same genome as everyone else, and acts locally — and out of billions of these myopic decisions a global body plan emerges, one robust enough to heal when injured.

The Distill article Growing Neural Cellular Automata distilled this idea into a delightfully small experiment: a grid of pixels, each running an identical learned rule that can only see its 3×3 neighborhood, trained so that a single seed pixel grows into a target image. This page repeats that experiment with a different kind of organism — the alphabet. Each letter above began as one live cell in the middle of an empty 32×32 grid. No cell knows where it sits, how big the grid is, or what an A looks like. There is no blueprint being copied and no coordinate system to consult; the shape is a negotiated agreement among neighbors. That the negotiation ends in a crisp glyph — and resumes correctly after you tear a chunk out — is the entire trick.

The model

Every cell carries a state vector of 12 numbers. The first three are RGB, the fourth is an “aliveness” alpha, and the remaining eight are hidden channels the cell may use however it likes — in practice they become an improvised chemical signaling system. A cell counts as alive when any cell in its 3×3 neighborhood has alpha above 0.1; everything else is dead and holds hard zeros.

One update step works like this. First, perception: each channel is filtered with a fixed identity kernel plus fixed Sobel-x and Sobel-y kernels, giving the cell a 36-number summary of its surroundings — its own state plus local gradients. That vector passes through two 1×1 convolutions: a dense layer with 64 units and a ReLU, then a linear layer back down to 12 numbers, initialized to zero so that the untrained network does nothing at all. The output is a residual update, added to the cell’s current state.

Two details do a surprising amount of work. Updates are stochastic: each cell fires with probability 0.5 per step, so there is no global clock to hide behind. And after every step the alive mask is re-applied, so a cell can only stay alive by keeping the neighborhood convinced. The whole rule is about five thousand parameters — one small MLP per letter, stamped onto every cell.

Training

Each character gets its own model. The target is that character rendered in DejaVu Sans Bold onto the 32×32 grid, and the loss is plain mean squared error between the target and the CA’s visible channels. What makes this trainable is that the rollout is differentiable: we run the automaton for a random 36–56 steps and backpropagate through every one of them, so gradient descent shapes not a picture but a process that ends in the picture.

Trained naively from the seed, the CA learns to pass through the target rather than stop at it — leave it running and the letter bloats into noise. The fix is the sample pool: instead of always starting from a seed, most training rollouts begin from the end state of a previous rollout. Old, already-grown patterns keep re-entering training, so any drift past the trained horizon gets punished, and persistence becomes part of the objective. To get regeneration, we go one step further and vandalize the pool: some samples have a disc of cells zeroed out before the rollout, and the model is graded on the repaired result. The same recipe also scales past single letters: the word models above are trained as one network with a row of coded seeds on one wide grid, and verified the same way — Tesseract has to read the entire grown word back from a single screenshot.

All of this was trained from scratch on a Raspberry Pi 5, CPU only. At five thousand parameters per letter, morphogenesis turns out to be a remarkably affordable hobby.

Does it actually write? OCR as a judge

A loss curve tells you the pixels are close to the target; it does not tell you the result reads as a letter. So the pipeline includes an independent judge with no stake in the outcome. After training, every letter is grown from a single seed for about 80 steps, screenshotted, and handed to Tesseract OCR — software that has never seen this project and only knows what letters are supposed to look like. A letter passes only if Tesseract reads back the character it was trained to be.

It is a pleasingly circular test: a neural network grows a glyph pixel by pixel, and a completely different piece of machine vision confirms the hallucination is legible. Try it

Scroll back up to the demo and be cruel to it. Drag a line straight through the middle of a letter: both halves will notice the wound, regrow toward each other, and knit back into the glyph, even though no cell has any idea a “whole letter” is the goal. Cut off a serifless limb, delete the counter of an O, or mash the damage button until the word is rubble — then just wait. The same local rule that grew each letter also maintains it, indefinitely. Growth, persistence, and repair were never three features; they are one behavior, observed at three moments.
